# Components

This document provides details on the available components in the Cashy Design System.

## Button

A versatile button component with multiple variants and states.

**Variants:**
- `primary` - Main call-to-action
- `secondary` - Alternative actions
- `tertiary` - Low-emphasis actions
- `danger-bold` - Destructive actions (high emphasis)
- `danger-subtle` - Destructive actions (low emphasis)

**Features:**
- Icon support (left/right)
- Loading states
- Disabled states
- Icon-only mode
- Three sizes (small, medium, large)
- Full accessibility support

## Toggle

A control for switching between two states (on/off).

**Features:**
- **Semantic Styling**: Uses `brand-solid` for active state and `tertiary` for inactive.
- **Sizes**: Small (`28px`), Medium (`44px`), Large (`56px`).
- **Label Positioning**: Supports Left or Right label placement.
- **Accessibility**: Built with hidden native input for full keyboard and screen reader support.

## Radio

A control for selecting a single option from a list.

**Features:**
- **RadioGroup**: Context-based wrapper for managing group state (controlled or uncontrolled).
- **Layouts**: Vertical (default) and Horizontal stacking.
- **Sizes**: Small, Medium, Large.
- **Rich Content**: Supports labels and helper descriptions.

## Checkbox

A control for selecting multiple options or binary states.

**Features:**
- **States**: Checked, Unchecked, and Indeterminate (for nested selections).
- **Sizes**: Small, Medium, Large.
- **Rich Content**: Supports labels and helper descriptions.
- **Interactive**: Fully interactive with `defaultChecked` support.
